# Opdracht 4 - Test software

## Testplan BakkerBartel

### Inleiding
Voor mijn project **BakkerBartel** heb ik handmatig getest of de belangrijkste functies gewoon goed werken.  
BakkerBartel is een webapp waarin werknemers vakantie kunnen aanvragen en beheerders die aanvragen kunnen bekijken, goedkeuren of afwijzen. Daarnaast kan de beheerder ook werknemers beheren.

Het doel van deze tests was om te checken of de belangrijkste onderdelen van de applicatie werken zoals ze moeten werken.

### Teststrategie
Ik heb gekozen voor **handmatig functioneel testen**.  
Dat betekent gewoon dat ik zelf door de applicatie heen ga in de browser en kijk of alles doet wat ik verwacht.

Ik heb vooral gekeken naar:
- werknemer kiest zichzelf en opent eigen aanvragen;
- beheerder opent kalender en aanvragenoverzicht;
- beheerder opent werknemerbeheer;
- foutieve invoer wordt tegengehouden;
- de belangrijkste pagina's laden gewoon goed.

### Testomgeving
- Windows
- ASP.NET Core Blazor Server
- SQL Server Express
- Browser: Chrome / Edge

### Doel van de test
Met deze test wilde ik controleren of de belangrijkste onderdelen goed genoeg werken voor gebruik en oplevering. Ik heb dus niet alleen gekeken of iets technisch werkt, maar ook of de pagina's duidelijk en bruikbaar blijven tijdens het gebruiken van de app.
