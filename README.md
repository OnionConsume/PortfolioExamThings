# Portfolio Examen - Overzicht

Dit portfolio is opgebouwd per opdracht. Elke opdracht heeft een eigen map met bewijs, toelichting en waar nodig extra bestanden zoals screenshots, video's of documenten.

## Structuur van het portfolio

### Opdracht 1 - Plant werkzaamheden en bewaakt de voortgang
In deze map staat bewijs voor:
- projectomschrijving;
- backlog en user stories;
- voortgangsbewaking;
- planning.

Belangrijke onderdelen:
- `1.1 Project omschijving/Projectomschrijving.md`
- `1.1-1.2-1.4 Backlog en voortgang/`
- `1.3 Planning/Planning BakkerBV.md`

### Opdracht 2 - Ontwerp software
In deze map staat het ontwerp van de applicatie, met onderbouwing en UML-diagrammen.

Belangrijke onderdelen:
- `Onderbouwing.md`
- `2.1 2,2/KlassenDiagram.jpg`
- `2.1 2,2/Sequence Diagram UML.png`

### Opdracht 3 - Realiseert onderdelen van software
In deze map staat bewijs van de gemaakte software, versiebeheer en werkende applicatie.

Belangrijke onderdelen:
- `Code/BakkerBartel-project.zip`
- `Code/readme.txt`
- `App uitvoeren/`
- `VersieBeheer/Branches/`
- `VersieBeheer/Commits/`
- `VersieBeheer/Merges/`

### Opdracht 4 - Test software
In deze map staat het testbewijs van de applicatie.

Belangrijke onderdelen:
- `Testplan.md`
- `Testscenario's.md`
- `Testrapport.md`

### Opdracht 5 - Doet verbetervoorstellen voor de software
In deze map staan de verbeterpunten die uit testen, oplevering en reflectie naar voren kwamen.

Belangrijk onderdeel:
- `Verbetervoorstellen.md`
- `Bewijs fixes/`

### Opdracht 6 - Voert overleg
In deze map staat bewijs van overleg, afspraken, standups en dagverslagen.

Belangrijke onderdelen:
- `Afspraken.md`
- `Daily_standups.md`
- `Sprint_meetings.md`
- `Bewijs/`
- `Dagverslagen/`

### Opdracht 7 - Presenteert het opgeleverde werk
In deze map staat de demo-presentatie van het project.

Belangrijk onderdeel:
- `presentatie.md`

### Opdracht 8 - Reflecteert op het werk
In deze map staat de retrospective met samenvatting en videolink.

Belangrijk onderdeel:
- `retrospective.md`

## Project zelf

Het project waar dit portfolio over gaat is **BakkerBartel**. Dit is een webapplicatie waarin:
- werknemers vakantie kunnen aanvragen;
- beheerders aanvragen kunnen bekijken, goedkeuren en afwijzen;
- werknemers beheerd kunnen worden;
- rekening wordt gehouden met planning en personeelsbezetting.
